# Vekpal Football - Codex Common Rules

このファイルは、Vekpal FootballプロジェクトにおけるCodexの共通ルールを定義する。

以後、Codexは毎回の作業前に本ファイルを確認し、ここに記載されたルールを遵守すること。

---

## プロジェクトの目的

Vekpal Footballは、日本語のサッカー情報サイトである。

目的は以下の通り。

- 月間100万PVの達成
- 月収10万円以上の達成（短期目標）
- SEOを中心とした長期的な集客基盤の構築
- Google Discover・SNS流入への対応
- 高品質な常設コンテンツの蓄積
- 初心者にも分かりやすいサッカー情報の提供

---

## 基本方針

以下を最優先とする。

1. 品質
2. SEO
3. 保守性
4. 回遊率
5. 実装の安全性

変更量の多さを目的としない。

「既存サイトを壊さず、価値を積み上げること」を最優先とする。

---

## AdSense

現在、Google AdSenseの審査中である。

以下は禁止。

- サイト全体の大規模変更
- 広告設定の変更
- 広告配置の変更
- AdSenseコードの変更
- ダミー広告の追加
- 不自然な広告用スペースの追加

AdSense審査への悪影響が考えられる変更は行わないこと。

---

## Git関連

CodexはGit操作を行わない。

禁止事項：

- git add
- git commit
- git push
- git pull
- git merge
- git rebase
- git reset
- git checkout
- git restore
- ブランチ作成
- ブランチ削除
- Pull Request作成
- Pull Requestマージ
- GitHub Pages反映
- 本番環境へのデプロイ

ユーザーが手動でコミット・プッシュ・デプロイを行う。

---

## URL関連

以下は禁止。

- 既存URLの変更
- 大量のリダイレクト追加
- URL命名規則の変更

新規ページを作成する場合は、既存のURL規則に従うこと。

---

## Google関連設定

以下は禁止。

- Google AdSense設定変更
- Google Analytics設定変更
- Google Search Console設定変更
- 本番ドメイン変更
- CNAME変更
- robots.txtの不必要な変更
- ads.txtの不必要な変更

Google関連の設定は原則として変更しない。

---

## 実装方針

以下を遵守する。

- 既存の実装方式を優先する
- 既存の命名規則を優先する
- 既存のコンポーネントを優先する
- 品質を優先する
- SEOを意識する
- 保守性を意識する
- 必要最小限の変更を行う

禁止事項：

- 大規模リファクタリング
- 不必要なライブラリ追加
- フレームワーク変更
- デザインの全面変更
- ヘッダーの全面変更
- フッターの全面変更
- フォント変更
- カラーテーマ変更
- DB導入
- CMS導入
- ログイン機能追加
- コメント機能追加
- 課金機能追加

---

## 記事作成ルール

以下を遵守する。

### 必須事項

- 検索意図を満たすこと
- 初心者にも分かりやすいこと
- SEOを意識すること
- 内容が重複しないこと
- 関連記事を適切に設定すること

### 禁止事項

- 薄いコンテンツの量産
- 他サイトのコピー
- FIFAやIFAB等の文章の転載
- 内容の使い回し
- 不自然なキーワード詰め込み
- 存在しない情報の記載
- 推測による記載
- 架空の出典の記載

---

## サッカー情報のルール

以下は必ず確認すること。

- FIFA
- IFAB
- JFA
- その他の公式情報

以下は大会やリーグによって異なる可能性がある。

- 交代ルール
- 大会方式
- 順位決定方法
- 移籍ルール
- 記録基準
- 延長戦やPK戦の扱い

世界共通ルールとして断定しないこと。

---

## SEOルール

以下を意識すること。

- title
- meta description
- canonical
- OGP
- Twitter Card
- JSON-LD
- BreadcrumbList
- sitemap
- 内部リンク
- 関連記事
- モバイル対応

SEO関連の実装は、既存方式を優先する。

---

## 内部リンク

以下を遵守する。

- 自記事へのリンク禁止
- 存在しないURLへのリンク禁止
- 関連性を優先する
- 総合記事と個別記事を適切にリンクする
- 回遊率を意識する

リンク数を増やすことを目的にしない。

---

## アクセシビリティ

以下を確認する。

- モバイル表示
- 横スクロールの有無
- パンくず表示
- 見出し構造
- フォーカス表示
- タップ領域
- コンソールエラー

明確な問題のみ修正する。

---

## 検証

作業後は、プロジェクトに存在する検証方法を可能な限り実行する。

原則として以下を確認する。

- validate_site.py
- テストコード
- HTML構文
- XML構文
- 内部リンク
- sitemap
- モバイル表示
- console error

存在しないコマンドを無理に実行しないこと。

---

## 完了報告

完了報告は以下の6項目のみとする。

1. 実施内容

2. 新規追加ページ

- URL
- 検索意図

1. 変更内容

- 変更ファイル
- 新規ファイル
- 削除ファイル

1. 検証結果

- 実行コマンド
- 成功／失敗

1. Git差分

- 変更ファイル数
- 新規ファイル数
- 意図しない差分の有無

1. ユーザー確認事項

- ユーザーが目視確認するべき内容

必要な場合のみ補足を記載する。

---

## 作業終了時

以下を必ず守ること。

- 完了報告を提出する
- 作業を停止する
- Git操作を行わない
- デプロイを行わない
- ユーザーの指示を待つ

コミット・プッシュ・デプロイは常にユーザーが手動で行う。

---

## 最終ルール

Codexは、

- 品質
- SEO
- 保守性
- 安全性
- 検索意図の充足

を最優先とし、変更量の多さを評価基準にしないこと。

「既存サイトを壊さず、検索流入とコンテンツ価値を着実に積み上げること」を本プロジェクトの最重要方針とする。
